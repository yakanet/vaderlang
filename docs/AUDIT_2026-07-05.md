# Audit complet du compilateur Vader — 2026-07-05

Audit multi-agents couvrant les 9 sous-systèmes : frontend (lexer/parser/fmt),
typecheck, resolver/modules/comptime, lower/midir, bytecode/VM, c_emit/runtime C,
stdlib, CLI/diagnostics/transversal, LSP. Chaque constat marqué **reproduit** a été
prouvé en exécutant `./build/vader` sur un snippet ; les repros vivent sous
`~/.claude/jobs/7a7575df/tmp/` (jetables). Les audits antérieurs
(`AUDIT_2026-06-03.md`, `AUDIT_2026-06-26.md`, `VADER_COMPILER_GAPS.md`) ont été
croisés : les statuts DÉJÀ CONNU y renvoient.

## Synthèse

Le compilateur est **globalement mûr pour un projet pre-MVP** : déterminisme du
bootstrap soigné (aucune fuite d'itération de map trouvée), registre de
diagnostics centralisé sans collision, SPEC.md fidèle, VM memory-safe, GC
natif (barriers, rooting, cards) vérifié sain, dedup de table de types corrigé,
LSP sans duplication de frontend. Les faiblesses réelles se concentrent en
quatre poches :

1. **Deux miscompiles critiques dans les passes midir** (invalidation
   incomplète dans `array_load_forward`, angle mort FieldGet/ArrayGet dans
   l'analyse d'échappement → pointeur pendant). Même signature que les bugs
   historiques de la zone (move-folding, StructNew) : analyses de flux sur des
   locals **mutables pré-SSA** avec des conditions de bail incomplètes.
2. **Un trou de soundness dans le typechecker** : le narrowing n'est jamais
   invalidé par une réassignation → programme mal typé accepté, miscompile
   natif silencieux. Deux frères : exhaustivité court-circuitée par les bras
   gardés, covariance des tableaux mutables.
3. **Le bord hôte des erreurs I/O est cassé** : le trait `Error` renvoyé par
   les intrinsics n'a pas d'identité de type → `.message()` crashe (exit 134)
   le compilateur lui-même (`vader dump /inexistant.vader`) et tout programme
   utilisateur.
4. **`vader fmt` et `std/regex` ne sont pas utilisables tels quels** : trois
   bugs de round-trip fmt qui corrompent du code stdlib réel ; trois crash/DoS
   regex sur entrée adverse.

Motif transversal le plus rentable à traiter : **la sémantique codée deux fois
qui diverge** (div/0 natif vs VM, string OOB natif vs VM, arithmétique comptime
i64 vs runtime i32, trois scanners lexicaux, deux calculs d'élément de
générateur, deux conventions d'erreur io vs reste de la stdlib).

## CRITIQUES (tous reproduits)

### CR1 — midir : `array_load_forward` forwarde une valeur périmée → miscompile VM + natif
`vader/midir/array_load_forward.vader:98-104` (+77-82) — NOUVEAU.
Sur `InstrArraySet`, la passe track `SlotVal{target, index, value}` où `value`
peut être un **local utilisateur mutable** ; `drop_slots_using` n'invalide que
si le tableau ou l'index est redéfini, jamais si `value` l'est.
Repro : `a[i]=x; x=x+5; y::a[i]` → `y=15` au lieu de `10`, **VM et natif**.
Dans le chemin de codegen réel (`cfg_to_bytecode`). Fix ~1 ligne : ajouter
`s.value == d` au filtre de `drop_slots_using`.

### CR2 — midir : l'analyse d'échappement ignore FieldGet/ArrayGet → pointeur pendant en natif
`vader/midir/escape.vader:229-300` — NOUVEAU.
`compute_escaping` ne modélise ni `InstrFieldGet` ni `InstrArrayGet` (`_ -> {}`).
Lire un sous-objet d'un conteneur non-échappant puis le retourner ne propage
pas l'échappement : le sous-objet est promu pile (`struct.new_stack`) et son
adresse sort de la fonction. Repro natif : `return outer.f` → valeur corrompue
(2 au lieu de 42) ; `return holder[0]` → garbage. La VM masque (stack≡heap).
Fix : `record_alias(result, target)` sur FieldGet/ArrayGet. Frère du segfault
StructNew corrigé — même famille, angle mort restant.

### CR3 — typecheck : le narrowing survit à la réassignation → miscompile natif silencieux
`vader/typecheck/stmt.vader:424` (`check_assign`) — NOUVEAU.
`check_assign` ne touche jamais `narrowed_bindings`/`narrowed_fields` : après
`x: i32|string = 5 ; if x is string { return } ; x = "now a string"`, `x` est
encore lu comme `i32`. Repro : `return x + 1` compile sans diagnostic, natif
imprime `2` (atome+1), exit 0 ; la VM trappe. Trois chemins prouvés (garde
divergente, in-block, champ narrowé). Le reset inter-passes existe
(`orchestrate.vader:871-876`) ; le trou est intra-passe. Fix : invalider
l'entrée narrow du symbole/champ dans `check_assign`.

### CR4 — bord hôte : `Error.message()` sur une erreur d'intrinsic I/O → crash exit 134, y compris du compilateur
`stdlib/std/io/io.vader:14` + `vader/vm/host.vader:262,278,310` +
`runtime/c/vader_runtime.c:2972` ; touche `vader/cli/main.vader:889,797` et
`bootstrap.vader:91` — NOUVEAU (deux agents indépendamment).
Les intrinsics fallibles (`read_file_bytes`, `write_file_bytes`, `read_dir`,
`read_line`, `read_stdin`) déclarent `| Error` (trait) mais l'hôte fabrique la
valeur **sans identité de type concret**. `is Error` marche ; `.message()` →
vtable miss → abort. Repro : `./build/vader dump --stage=lexer /nope.vader` →
« reached unreachable at vtable miss in Error.message », exit 134. Idem VM.
Corollaire stdlib : `if r is IOError` est une **erreur de compilation** T3040
(IOError n'est jamais une valeur de `Error | string`) — il n'existe aucun moyen
de lire le message d'une erreur io. Fix : retourner `IOError` concret dans les
unions (la parade est déjà documentée dans `string.vader:550`) + matérialiser
l'identité de type au bord hôte. Arbitrage archi recommandé (VM + runtime C).

### CR5 — std/regex : trois crash/DoS sur entrée adverse
`stdlib/std/regex/regex.vader` — NOUVEAU (tous reproduits).
- **ReDoS exponentiel** (`:685,772`) : `add_thread` sans ensemble « visited »
  par pc — l'invariant central du Thompson NFA manque. `(a+)+c` : N=16→1,8s,
  N=22→timeout. L'en-tête « O(n×m), no catastrophic backtracking » est faux.
- **Bombe mémoire `{m,n}` inversé** (`:328`) : `diff :: m - n` en usize sans
  vérifier `m>=n` ; `compile("a{2,1}")` → 5,4 Go RSS puis SIGABRT.
- **Cycle epsilon** (`:772-813`) : `(a*)*` → stack overflow VM, segfault natif.
Un set visited par pc répare ReDoS **et** cycle epsilon ; ajouter `m>=n` + un
plafond de taille de programme (`a{999999999}` → OOM aussi, cf. M-stdlib).

## MAJEURS

### Typecheck / soundness
- **T1 — Bras de match gardé compte pour l'exhaustivité** — NOUVEAU, reproduit.
  `match_expr.vader:86-130` : la couverture s'applique sans tester `arm.guard`.
  `is i32 as n if n > 0` + `is string` jugé exhaustif → `pick(-3)` atteint
  unreachable (exit 134). Fix : ne pas soustraire/poser `has_wildcard` quand
  `arm.guard != null`.
- **T2 — Covariance des tableaux mutables** — NOUVEAU, reproduit.
  `type_assign.vader:125-134` : `i32[]` assignable à `(i32|string)[]` mutable →
  corruption VM / recopie silencieuse natif (les deux backends divergent entre
  eux et de SPEC:623). Fix : invariance des éléments pour cible mutable.
- **T3 — Casts non validés** — DÉJÀ CONNU (différé Phase 5.6) : `i32("hello")`
  typecheck, natif miscompile silencieusement. Reclassé majeur : silencieux.

### Lower / générateurs
- **L1 — `break`/`continue` étiquetés dans un générateur ignorent leur label**
  — NOUVEAU. `lower_generator.vader:257-263` : `loop_stack[len-1]` systématique,
  `LoopFrame` sans champ label (contraste `build.vader:483-492`
  `resolve_loop_frame`). `break outer` cible la boucle interne → miscompile.
- **L2 — `gb_route_stmt` ne route pas `LoweredCellSet`/`LoweredDeferPush`**
  — NOUVEAU. `lower_generator.vader:508` : le `_ ->` republie sans réécrire
  vers `self.<field>` (alors que `gb_route_expr` est exhaustif et gère
  cells/closures). Un local spillé réassigné via cellule, ou un `defer` dans un
  générateur → référence pendante. Même angle mort dans la collecte de spill
  (`:449,462`).
- **L3 — IR verifier designé mais non implémenté** — DÉJÀ CONNU, toujours
  ouvert. Seule couverture : panics ad hoc dans `midir/emit.vader`. Note : CR1
  et CR2 ne seraient pas attrapés par le verifier tel que designé (IR
  structurellement valide, sémantiquement fausse) — il faut AUSSI des tests
  ciblés des passes d'optim sur les motifs mutable-local / escape.

### Backend C / divergences natif↔VM
- **B1 — `@extern` avec paramètre string ne compile plus** — NOUVEAU, reproduit.
  `c_emit/host.vader:196-198` : le shim marshalle `a.ptr`/`a.len` de l'ancien
  fat-pointer ; `vader_string_t` est un atome u32 depuis 7eb041a8 (2026-05-25).
  `tests/snippets/extern_native_basic` → cc failed. **Régression de ~6 semaines
  non captée** → trou de couverture du chemin natif @extern. Fix :
  `vader_atom_len/data` + `vader_string_to_cstr`.
- **B2 — Division/modulo par zéro : trois comportements** — NOUVEAU/PARTIELLEMENT
  CONNU (stance SPEC:99), reproduit. VM trappe proprement ; natif arm64 renvoie
  silencieusement 0 (et `INT_MIN/-1` wrappe) ; x86 SIGFPE. Le commentaire
  `walker.vader:629-631` (« a trap fires ») est faux sur arm64. Recommandation :
  garde `r==0 → vader_trap` pour aligner sur la VM, ou a minima corriger le
  commentaire et documenter la divergence.
- **B3 — Constante f64 ±Inf/NaN émet du C invalide** — DÉJÀ CONNU (TODO dans le
  code), reproduit. `walker.vader:508` : `1.0e308*100.0` comptime-foldé →
  `inf.0` → cc failed. Fix : `(1.0/0.0)` etc.
- **B4 — Indexation string codepoint OOB : VM rend U+0000, natif trappe**
  — NOUVEAU, reproduit. `vm/host.vader:157-170` (le commentaire l'admet) mais
  absent de `KNOWN_DIVERGENT`. Grave au **comptime** : `s[i]` OOB bake
  silencieusement U+0000.

### Comptime / reproductibilité
- **CT1 — Arithmétique comptime tree-walk en i64 sans largeur** — NOUVEAU,
  reproduit. `comptime/eval_binary.vader:93-113` : `@comptime (2e9+2e9)>0` bake
  `true`, le runtime i32 donne `false`. Le chemin VM-backed, lui, wrappe
  correctement → la même const peut diverger selon le chemin d'évaluation.
- **CT2 — Hôtes impurs servis au comptime sans gate** — DÉJÀ CONNU (mémoire),
  confirmé bout-en-bout. `vm/host.vader` sert TimeNow/Env/IoRead/Spawn sans
  consulter `is_comptime` ; `@comptime BUILD_TS: i64: ts()` bake un timestamp
  mural sans diagnostic. `resolve_vm_required_decls` est dans le pipeline CLI
  principal : un `@comptime` impur dans `vader/` casserait stage1==stage2 en
  silence. Fix peu coûteux : gate `is_comptime` sur les hôtes non-purs
  (nouveau C4xxx).
- (Mineur associé : comparaisons d'ordre `<` string/bool → toujours `false` au
  comptime tree-walk, `eval_binary.vader:228-244`.)

### Frontend / fmt
- **F1 — Littéral entier hors borne u64 silencieusement wrappé** — NOUVEAU,
  reproduit. `lexer.vader:349-384` : 2^64 → 0, 32 chiffres → garbage, aucun
  diagnostic. Miscompile silencieux.
- **F2 — BOM UTF-8 rend un fichier irrésolvable** — NOUVEAU, reproduit.
  `parser.vader:67-125` : `parse_module_header` ne saute pas le BOM
  (contrairement au lexer) → R2001/R2020 trompeurs. Cause racine : **trois
  scanners lexicaux parallèles** (tokenize, parse_module_header,
  fmt/comments.vader) qui divergent.
- **F3/F4/F5 — `vader fmt` corrompt du code valide** (trois bugs de round-trip,
  NOUVEAUX, reproduits) :
  - `implements Trait<Nested<T>>` → `>` en trop → non-compilable (span du `>>`
    splitté, `parser.vader:265-273` + `printer.vader:726,1674`). Frappe
    `core.vader:307`, `string.vader:181`, `collections.vader:379`.
  - Le sucre `NAME :: @comptime <expr>` est réécrit en fonction synthétique
    `__comptime_NAME` (aucun marqueur synthetic sur le decl désucré,
    `parser.vader:1372-1418`). Corromprait `dtoa_tables` (8×) et `crypto`.
  - Champ-littéral de struct-pattern reformaté `:` → `=` qui ne reparse plus
    (`printer.vader:1779-1782`).
  → Une passe de fuzz round-trip (format→reparse→compare) sur tout l'arbre
  s'impose avant de considérer fmt utilisable.
- **F6 — SIGSEGV sur imbrication profonde** — NOUVEAU, reproduit. ~900 niveaux
  de parens (ou `!`×10000, `[]`×10000) → exit 139 sans message, parseur récursif
  et walkers sans limite de profondeur.

### Stdlib (hors regex/io déjà en critique)
- **S1 — JSON : récursion non bornée** — NOUVEAU, reproduit. `json.vader:133` :
  512 `[` → stack overflow au lieu d'un JsonError.
- **S2 — JSON : surrogates `\u` non recombinées** — NOUVEAU, reproduit.
  `json.vader:298-311` : U+1D11E échappé → 6 octets WTF-8 au lieu de 4 octets

- **S3 — numbers : overflow silencieux `parse_int_in_base`/`parse_uint_in_base`**
  — NOUVEAU, reproduit (`string::parse_int`, lui, checke — modèle à généraliser).
- **S4 — time : overflow i64-ns silencieux** — NOUVEAU, reproduit.
  `parse_iso8601("9999-…")` → reformate en `1816-03-30…` sans TimeError, alors
  que la doc annonce « after 9999 ».
- **S5 — semver : overflow i32 dans les bornes de ranges** — NOUVEAU, reproduit.
  `^2147483647.0.0` n'accepte plus rien (wrap du `+1`). Et `>= 1.2.3` (espace,
  forme npm courante) rejeté.
- **S6 — regex `{n}` sans plafond** (bombe compile) et **`matches` dépendant de
  l'ordre des alternatives** (`a|ab` vs `ab|a`) — NOUVEAUX, reproduits.
- **S7 — crypto : pas de digest brut u8[], pas de HMAC, length-extension non
  documenté** — NOUVEAU. Les hashes eux-mêmes sont bit-exacts (15 vecteurs vs

## MINEURS (sélection, regroupés)

- **Frontend** : CR seul (`\r`) n'incrémente pas les lignes (diags en ligne 1) ;
  interpolation `"${` non terminée sans diag lexer (P1003 vague) ; overflow
  float silencieux (`1e400` → inf) ; `pop_interp` FIXME.
- **Resolver** : doublon fn/non-fn détecté dans un seul ordre
  (`collect.vader:101-151`) ; `@internal` sur une fn → R2010 « decorator
  unknown » trompeur ; walkers `substitute_body` @partial qui aliasent l'AST du
  trait (landmine si un défaut de trait construit `T{}`/lambda) ;
  `StampCtx.base_line` collisionne au-delà de 1024 colonnes.
- **VM/bytecode** : `MAX_FRAME_DEPTH` 1024 (récursion légitime OK natif, trap
  VM/comptime) ; format `.virt` sans magic/version, lignes inconnues droppées
  silencieusement (PC décalés) ; `INT_MIN/-1` non gardé ; u64 > i64.MAX affiché
  signé dans struct/array (latent).
- **Runtime C** : `vader_read_line` fgets 4096 (ligne longue coupée) ;
  `ftell/long` (fichiers >2 Go Windows) ; registre vtable sans plafond vs
  `VADER_MAX_TRAIT_SLOTS=64` (>~56 méthodes trait → C invalide) ; collision de
  mangle possible via `sanitise` (`a$b`/`a_b`) ; `emit_slot_*_boxed` sans bounds
  check (code mort, hazard latent).
- **Stdlib API** (cohérence) : quatre conventions d'erreur (`null` / type
  concret / trait cassé / sentinelle) ; `string::find` retourne `bool` (c'est
  un `contains`) vs `iter::find`/`regex::find` ; quatre verbes pour « ajouter »
  (push/add/put/append) ; `ParseError` retourné par numbers/semver sans
  ré-export ; `parse_int`→i32 vs `parse_int_in_base`→i64 ; `parse_f64`→null vs
  `parse_float`→ParseError ; `math::min/max/clamp` couverture inégale ;
  `byte_substring` toujours absent (DÉJÀ CONNU).
- **Docs mensongères vérifiées** : `math::min` NaN (« propagates » — faux,
  dépend de l'ordre) ; SPEC `to_hex` « lowercase » (émet UPPERCASE) ; SPEC
  `decode_escapes` `\u{HHHH}` non implémenté ; vaderdoc `numbers::to_hex` avec
  suffixe `255_u64` supprimé (ne compile plus) ; vaderdoc égarée au-dessus de
  `module` (pi, IOError) ; `time::parse_iso8601` accepte garbage après `Z` et
  `2026-02-30` ; JSON stringify Inf/NaN → JSON invalide.
- **Diagnostics/CLI** : fichier introuvable confondu avec R2001 ; rendu minimal
  (pas de caret, Span.end jamais utilisé, pas de fix-it) ; pas de cap/dédup de
  diags ; `main.vader` monolithe 1429 LoC ; midir/ et c_emit/ sans aucun test
  colocalisé.
- **LSP** : Content-Length malformé → exit ; pas de borne sur Content-Length ;
  `binding_hv_walk_expr` ne descend pas scrutinee/garde de match (walker oublié
  du fix 06-26).
- **Lower** : `generator_element_of` (lower) vs `iterator_element_of`
  (typecheck) et vs `genstate_element` (fusion) — trois calculs parallèles de
  l'élément d'un `Iterator<T>` ; branches `"std/…"` littérales dispersées sur
  ~20 fichiers (hack lower_populate, connu) ; `const_fold_project` seulement
  dans le chemin dump (dump non représentatif du codegen).
- **Typecheck** : pas de contrôle de plage des littéraux entiers compile-time
  (`a: i32 = 9000000000` tronque en silence — incohérent avec char/T3055) ;
  gardes sur motif `.Variant` non supportées (cascade d'erreurs de parsing).

## Vérifiés sains (ne pas re-suspecter)

- **GC natif** : write barriers présents sur tous les chemins vivants, rooting
  shadow-stack complet, `compute_may_alloc` conservateur, protocole de cards
  correct, overflows d'alloc gardés. Aucune corruption mémoire reproduite.
- **Déterminisme bootstrap** : émission par tableaux (pas de MutableMap),
  sorties triées, BFS d'imports stable ; `verify.sh` garant.
- **VM** : bounds-checks exhaustifs, match d'opcodes exhaustif (166 bras, sans
  `_ ->`), dispatch host trap-on-miss, comptime borné (step-budget 100M,
  Trap→C4002 propre).
- **cfg_dce/fold_moves** : bail lecture ET écriture — le bug historique
  swap/reassign est corrigé, pas de frère. Liveness/DIE corrects. Court-circuit
  `&&`/`||` toujours branché (le prédicat tuple-pattern est sûr).
- **Dedup table de types** : clé unifiée `type_intern_key`, placeholder→upgrade
  récursion-safe — le bug BcStruct/BcRef ne peut plus se former.
- **Lexer** : itératif, progression garantie, robuste (binaire aléatoire, UTF-8
  cassé, recovery borné). Correctifs fmt de précédence 06-26 présents.
- **LSP** : réutilise le vrai lexer/parser (pas de tables dupliquées ; bug
  `yield` corrigé des deux côtés), JSON-RPC robuste, capabilities cohérentes,
  buffer-stale corrigé (doc_overlay).
- **Codes de diagnostic** : registre centralisé typé par phase, zéro collision.
- **SPEC.md** : 10 affirmations échantillonnées, aucune dérive réelle.
- **Stdlib** : crypto bit-exact ; base64 corrigé (bug 06-03 résolu) ; random
  (xoroshiro128++, rejection sampling), sort (stable), path, cli, ppm propres ;
  semver sans panic et précédence §11 correcte.
- **Comptime** : boucle infinie/panic comptime → diagnostic propre, jamais un
  crash du compilateur. Cycles d'imports et de deps @comptime détectés
  proprement. Plus aucun map projet-global indexé par node-id.
