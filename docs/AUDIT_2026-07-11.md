# Audit complet du compilateur Vader — 2026-07-11

Audit multi-agents (workflow `compiler-audit-2026-07-11`, 122 agents, 0 erreur,
~6,9 M tokens) couvrant 18 shards : `lexer`, `parser`, `fmt`, `typecheck-core`,
`typecheck-types`, `resolver-modules`, `comptime`, `lower-desugar`, `lower-mono`,
`midir`, `bytecode`, `vm`, `c-emit`, `stdlib-core`, `stdlib-domain`,
`cli-diag-dap`, `lsp`, `syntax-spec-dx`. Chaque shard a chassé six catégories —
incohérences, duplications regroupables, incongruités de syntaxe, bugs,
diagnostics manquants, features DX — puis chaque constat est passé par une
**vérification adversariale** (skeptique, `REFUTED` par défaut) :

- les **bugs** et **diagnostics manquants** sont prouvés par un snippet compilé
  et exécuté sur `./build/vader` **en VM (`run`) et en natif (`build`)** ;
- les constats **non-bug** sont re-lus contre le code cité et croisés avec
  `SPEC.md`.

**Résultat brut** : 104 constats → **89 CONFIRMED**, 5 PLAUSIBLE, 10 REFUTED.
Répartition des confirmés : 11 bugs critiques, 8 bugs majeurs, 8 diagnostics
manquants majeurs, 12 incohérences majeures, 1 duplication majeure, + ~55
mineurs. Les snippets de repro (jetables) ont été écrits sous le scratchpad de
session ; les faits `file:line` renvoient à l'arbre au 2026-07-11.

## Synthèse

Le compilateur reste **mûr pour un pre-MVP** : les quatre CRITIQUES de l'audit
2026-07-05 (CR1 `array_load_forward`, CR2 escape FieldGet/ArrayGet, CR3
narrowing survivant à la réassignation, CR4 `Error.message()` crash) **semblent
toutes corrigées** — aucun agent n'a pu les reproduire sur le binaire courant
(détail plus bas). Les faiblesses réelles de cette passe se concentrent en cinq
poches :

1. **Backend natif : divergences silencieuses VM↔natif** — `u32→f64` sign-extend
   en natif (CR-a), `array.slice` absent de `op_allocates` (pas de frame GC →
   pointeur pendant, **régression d'un point « vérifié sain » en 07-05**), byte
   Buffer non borné en natif, `is <largeur-int>` divergent. Même famille que les
   deux-sémantiques historiques.

2. **Typecheck : trous de soundness sur les opérateurs** — `==`/`<` ne vérifient
   *jamais* la compatibilité des opérandes (struct==primitif, familles
   numériques mixtes, RHS d'impl `Equals`/`Comparable` non typé), les casts ne
   sont **jamais** validés (`i32("hello")` compile), les littéraux entiers hors
   plage passent, la covariance des conteneurs génériques mutables reste ouverte
   (T2). Tous mènent à un ICE de lowering ou un miscompile natif, pas à un
   diagnostic.

3. **`?` (try) globalement non contraint** — saute le drain des `defer` sur le
   chemin d'erreur (**y compris en async**), n'exige pas que la fn puisse
   propager une `Error` (SIGSEGV du compilateur), ne vérifie pas la
   compatibilité du type de retour.

4. **fmt casse le round-trip sur `?`/`await`** — trois bugs de parenthésage
   (`(a+b)?`, `(await x)?`, `(await x).f`) réécrivent du code valide en code qui
   re-parse différemment. `fmt` reformate aussi les bases numériques d'enum et
   embarque un 3ᵉ scanner lexical (F2).

5. **stdlib : crashs sur entrée adverse** — `std/regex` (deux DoS distincts, dont
   le sibling de CR5), `std/string::compare_ascending` (OOB sur multibyte →
   crash de `sort`), `std/json` stringify de NaN/Inf (SIGSEGV).

Le motif transversal le plus rentable reste **la même sémantique codée deux fois
qui diverge** et **les `match`/if-chains sur unions fermées sans garde
exhaustive** (midir `build_stmt`/`emit_instr`, `type_contains_param`, les trois
walkers de substitution). Deux tables synchronisées à la main réapparaissent
partout (mangle-suffix, primitive-tag scanners, containment LSP, renderers de
diagnostics).

## CRITIQUES (11, tous reproduits en VM **et** natif sauf mention)

### CR-a — c-emit : conversion `u32 → f64` sign-extend en natif, zero-extend en VM
`vader/c_emit/walker.vader:695-698` + `vader/vm/value.vader:188`.
`emit_convert` ne zero-extend une source unsigned étroite que si la **cible** est
`I64`. Pour `u32 → f64` la cible est `F64`, donc la source reste `.I32` et on
émet `((double)(int32_t) x)`. Un `u32 ≥ 2³¹` devient un double **négatif**. Le VM
normalise toute source unsigned en amont (`convert_value`) → valeur correcte.
Repro : `x: u32 = 3000000000 ; f64(x) > 0.0` → VM `pos`, natif `neg`.
Le C émis contient `(double)(int32_t) INT32_C(-1294967296)`.
**Fix** : router la primitive via `uint32_t` aussi pour la cible `F64`.

### CR-b — c-emit : `ArraySlice` absent de `op_allocates` → pas de frame GC → pointeur pendant
`vader/c_emit/walker.vader:149-163` + `runtime/c/vader_runtime.c:2537`.
`op_allocates` liste `ObjectNew/ArrayNew/ArrayPush/…` mais **pas** `vm.ArraySlice`,
alors que `vader_array_slice` appelle `vader_gc_alloc` inconditionnellement. Une
fn dont la seule op allouante est `array.slice` est classée `no_frame` : ni
prologue shadow-stack, ni rooting des params/locals ref. Un GC déclenché par le
slice déplace les objets vivants sans mettre à jour les slots → lecture d'un
header périmé. Le code appelle pourtant lui-même `array.slice` « the GC-moving
array.slice » dans `is_cache_safe` — **incohérence interne** et **régression du
point « rooting shadow-stack complet » déclaré sain en 07-05**. Repro : VM
correct, natif corrompu sous pression GC.
**Fix** : ajouter `if op is vm.ArraySlice { return true }` à `op_allocates`.

### CR-c — fmt : `emit_try` supprime les parenthèses d'un opérande `?` non-postfix
`vader/fmt/printer.vader:1770`.
`emit_try` émet `self.emit_expr(x.inner)` sans garde de précédence puis appose
`?`. Comme `?` lie à bp 100 (plus serré que tout binaire/préfixe), `(a + b)?`
→ `a + b?` = `a + (b?)`, et `(await x)?` → `await x?` = `await (x?)`. Violation
de la garantie d'idempotence de `format_source`. Repro : `(await fetch(x))?`
reformaté donne un AST différent (ordre `AwaitExpr`/`TryExpr` inversé).
**Fix** : router via `emit_postfix_target` (comme `.field`/`[i]`/appel).

### CR-d — fmt : `postfix_target_needs_parens` omet `AwaitExpr`
`vader/fmt/printer.vader:1356` (+ `emit_await` non gardé à `:1775`).
`(await x).f` → `await x.f` = `await (x.f)` : le champ/index/appel se re-lie sous
le `await`. Miroir dans `emit_await` : `await (a + b)` → `await a + b`. Repro :
`(await make()).value` reformaté → programme rejeté à T3008 alors que
l'original type-checke.
**Fix** : ajouter `if e is p.AwaitExpr { return true }` + garder l'inner de
`emit_await`.

### CR-e — lower : `expr?` en propagation d'erreur saute le drain des `defer`
`vader/lower/lower_try.vader:61,87`.
`lower_try` fabrique le retour anticipé comme un `LoweredReturn` brut, **sans**
passer par `drain_all_defers` (contrairement à un `return` source). Les `defer`
du scope englobant ne sont pas exécutés sur le chemin d'erreur — **pire en async**
(le drain per-frame `LoweredDeferPopExec` est aussi contourné). Repro : `defer
println("cleanup")` puis `might_fail()?` sur le chemin d'erreur → « cleanup »
**absent**, VM et natif.
**Fix** : router le retour synthétique par `lower_return` (drain inclus).

### CR-f — resolver : clone de méthode-trait par défaut → `Self`/type-params non substitués + mutation de l'AST partagé du trait
`vader/resolver/substitute_body.vader:47,169,315`.
`substitute_expr_full` est un `@partial` dont le `_ -> return e` laisse passer
`StructLitExpr`/`LambdaExpr`/`SeqLitExpr`/`RangeExpr`/`TryExpr`/`CastExpr`/… **inchangés** :
(a) un `Self`/`T` niché dedans n'est jamais réécrit ; (b) l'arm renvoie le nœud
**original** (aliasing), et `assign_block_ids` réassigne ensuite ses `id` in-place
→ corruption des node-id du trait partagé par tous les impls. Repro : trait avec
défaut `wrap :: fn(self) -> Box = Box { .v = self.tag() + 100 }`, deux impls →
défaut miscompile (VM exit 0 muet, natif `unreachable` exit 134) vs impl explicite
qui imprime `101 102`.
**Fix** : compléter `substitute_expr_full` avec des arms de clonage réels
(node frais, `UNASSIGNED_NODE_ID`) ; jamais de fall-through aliasé.

### CR-g — stdlib : `compare_ascending` indexe `s[i]` en octets alors que `s[i]` est codepoint-indexé → OOB
`stdlib/std/string/string.vader:139-150`.
Borne de boucle `n = min(a.bytes().len(), b.bytes().len())` (octets) mais lecture
`a[i]`/`b[i]` **codepoint-indexée** (SPEC.md:624). Sur toute chaîne multibyte le
nombre de codepoints < octets → `a[i]` **trap** (OOB dur, pas valeur fausse). Le
sibling `compare_ascending_ci` le fait correctement via vues octets. Utilisé par
`sort(strings, compare_ascending)` → **trier un tableau contenant un accent/CJK
crashe**. Repro : `compare_ascending("été", "été")` → trap VM et natif.
**Fix** : comparer `a.bytes()[i]`/`b.bytes()[i]` (l'ordre UTF-8 octet = ordre
codepoint).

### CR-h — stdlib : `std/regex` compile sans garde de profondeur → stack overflow (DoS)
`stdlib/std/regex/regex.vader:406-411`.
Le parseur regex est une récursion mutuelle dont la profondeur = profondeur
d'imbrication des groupes. Contrairement à `std/json` (garde `MAX_DEPTH=200`
après S1), **aucun plafond** : ~1100 `(` dépassent la limite de 1024 frames du VM
(trap) et segfaultent en natif au lieu de retourner `RegexError`. Reachable depuis
tout pattern utilisateur → DoS.
**Fix** : threader `depth` dans `parse_alternation`/`parse_concat`/… comme json,
`RegexError` au-delà de `MAX_DEPTH`.

### CR-i — stdlib : `std/regex` match — clôture epsilon `add_thread` récurse en O(longueur programme)
`stdlib/std/regex/regex.vader:819-833` (prior **CR5**, chemin distinct).
Le dédup `seen[pc]==gen` borne le nombre de pc distincts (fix du cycle epsilon
`(a*)*`) mais **pas la profondeur** : une chaîne acyclique de N `a?` compile en N
`Split` chaînés → match récurse ~N. Aucun plafond `PROGRAM_MAX` hors
`apply_counted`. Repro : 1500 × `a?` → trap VM (stack overflow 1024), divergence
VM↔natif.
**Fix** : clôture epsilon en worklist itératif, ou `PROGRAM_MAX` sur tous les
chemins compile.

### CR-j — typecheck : covariance des type-args de conteneur générique **mutable** (unsound, aliasing)
`vader/typecheck/type_assign.vader:225-236` (prior **T2**, moitié struct).
`struct_args_compatible` vérifie chaque type-arg via `is_assignable` **une seule
direction** (covariant), pour tout `StructType`, sans égard à la mutabilité — le
commentaire l'admet « knowingly unsound … deferred ». `MutableMap<K, i32>` est
assignable à `MutableMap<K, i32 | string>` ; une écriture par l'alias élargi
corrompt le backing partagé. La moitié *tableau* de T2 a été corrigée
(`is_assignable_array`), la moitié *conteneur générique* reste ouverte. Repro
(`typecheck-core-3` + confirmé par `typecheck-types-3`) : `n.put(0, "hi")` puis
`m.get(0)` relu en `i32` → VM trap, natif imprime un atom-id corrompu.
**Fix** : invariance pour les args des conteneurs mutables (bidirectionnel),
comme `is_assignable_array`.

### CR-k — typecheck : `==`/`!=` ne vérifient **aucune** compatibilité d'opérande
`vader/typecheck/binary.vader:243-255`.
`check_equality` n'exige un impl `Equals` que si **les deux** opérandes sont des
struct, et ne valide jamais la compatibilité des types (contrairement à
`check_arith`). `structValue == 5`, `structA == structB`, `enumDir == enumColor`
type-checkent tous. Pour `p == 5`, le lowerer émet `equals(p, 5)` → l'entier 5
lu comme pointeur `Point`, ou ICE midir. Le check T3040 n'est câblé que pour
l'opérande `null`. Repro : `Point{.x=1} == 5` → panic midir exit 134, VM et
natif, **sans diagnostic typecheck**.
**Fix** : porte de compatibilité symétrique dans `check_equality` (et
`check_is_overlap` pour opérandes concrets, pas seulement `null`).

## MAJEURS

### Backend natif / divergences VM↔natif

- **c-emit-3 — byte Buffer natif non borné vs VM qui trappe.**
  `runtime/c/vader.h:578-585` fait `b->bytes[off]`/`memcpy` sans check ; le VM
  passe par `check_byte_range` (`vader/vm/exec.vader:2357`) qui trappe, et les
  ops de slot tableau en natif ont pourtant la garde D7. Accès Buffer OOB → trap
  déterministe en VM, UB/corruption silencieuse en natif. (Buffer interne stdlib,
  pas directement user-reachable, mais tout bug d'offset stdlib corrompt en
  silence.) **Fix** : garde dans `vader_buffer_load_*`/`store_*`.

- **vm-2 — `is <largeur-int>` sur valeur boxée renvoie vrai pour la mauvaise
  largeur.** `vader/vm/value.vader:570-587`. `primitive_matches` renvoie
  `v is I32Val || v is I64Val` pour toute classe int suffixée → un `i32` boxé
  satisfait `is i64`. Le natif compare le tag exact → faux. Repro :
  `v: i32|i64 = 42 ; v is i64` → VM `WRONG-i64`, natif `ok-not-i64`. Mine aussi
  la sélection d'arm `match` discriminant des largeurs. **Fix** : porter la
  largeur source sur la Value / résoudre le ValType comme le natif.

### Typecheck / soundness

- **typecheck-core-1 — comparaison entre familles/largeurs numériques
  mélangées, pas de T3017.** `vader/typecheck/binary.vader:184-255`.
  `check_compare` renvoie `bool` dès que `is_numeric(left) && is_numeric(right)`,
  `check_equality` pour deux non-struct quelconques — aucun check de famille ni de
  largeur, alors que le header du module promet T3017. `f64 < i32`, `i32 < i64`,
  `u32 == i32` compilent. Repro : `x: f64 = 3.0 ; y: i32 = 5 ; x < y` → **VM trap
  `f64.lt`, natif imprime `less`** (divergence). **Fix** : réutiliser
  `numeric_result` (nom + largeur) avant `TY_BOOL`.

- **typecheck-core-2 / typecheck-types-2 — `Comparable`/`Equals` ne vérifient pas
  le RHS contre l'impl.** `vader/typecheck/binary.vader:194,251`. `check_compare`
  renvoie `bool` dès que le **gauche** a un impl `compare`, sans vérifier que le
  droit est assignable au param `other: Self` (`check_arith` le fait). `foo < bar`
  / `foo == bar` entre deux structs différents type-checkent → ICE midir. Repro :
  `Money < 5`, `Foo == Bar` → panic midir exit 134. **Fix** : valider
  `is_assignable(right, substitute_self_in(param0, left))` (miroir de
  `check_arith`).

- **typecheck-types-4 — les casts ne sont **jamais** validés.** `vader/typecheck/
  expr_cast.vader:19-23` (prior **T3**). `infer_cast` renvoie la cible
  inconditionnellement ; `i32("hello")`, `bool(struct)` passent. T3010 existe dans
  le registre, jamais émis. Repro : `i32("hello")` → VM trap convert, natif
  imprime du garbage, aucun diagnostic. **Fix** : tabuler les paires de conversion
  légales, émettre T3010 sinon.

- **syntax-spec-dx-1 / typecheck-core-4 — littéral entier hors plage compile en
  silence.** `vader/typecheck/type_ops.vader:250`, `type_assign.vader:184`.
  `narrow_for_expected` épingle un `FreeInt` dans un slot typé sans vérifier qu'il
  rentre (asymétrique avec le char, borné par T3055). Repro : `x: i32 =
  9000000000` → `410065408`, `z: u32 = -1` → `4294967295`, exit 0 sur les deux
  backends. **Fix** : check `int_fits(value, name)` au point d'épinglage (réutiliser
  la machinerie `char_fits`/T3055), gérer `UnaryExpr(.Neg, IntLit)`.

### `?` (try)

- **lower-desugar-2 — contrat de propagation `?` non vérifié.**
  `vader/lower/lower_try.vader:87`, `typecheck/expr_try.vader:19`. `infer_try`
  ne vérifie pas que le type de retour de la fn peut contenir la variante d'erreur
  propagée ; `lower_try` bâtit le retour sur l'union complète en contournant la
  coercion de `lower_return`. Repro : `parse()?` (`i32 | PErr`) dans une fn
  `-> string` → compile sans diagnostic, VM trap type-confusion. **Fix** : vérifier
  dans `infer_try` + router par `lower_return`.

- **syntax-spec-dx-6 — `?` utilisable dans une fn qui ne peut pas propager
  d'`Error`, et sur une expression non-faillible.** `vader/typecheck/
  expr_try.vader:19-27` (SPEC.md:1818 l'admet « not yet enforced »). Repro :
  `read_file_string(path)?` dans une fn `-> string` → **SIGSEGV du compilateur
  (exit 139)** VM et natif, aucun diagnostic. **Fix** : exiger une variante
  `Error` dans l'inner et threader le type de retour englobant.

### Frontend / parser / fmt

- **parser-1 — `if x is Foo<T> { … }` mal-parsé.** `vader/parser/parser.vader:
  2988,3020`. En condition (`allow_struct_lit=false`), `scan_angle_generic`
  s'engage sur une instanciation générique et attend un `(`, mais rencontre le `{`
  du corps → erreur P1002. Le même test marche en arm `match` (route par
  `parse_type`). Repro : `if x is Box<i32> { return 1 }` → échec de parse VM et
  natif. **Fix** : faire parser le RHS de `is` par `parse_type` (résout aussi
  parser-2). *(Voir parser-2 pour la variante `const T[]`/`fn(...)`.)*

- **parser-3 — imbrication de *statements* (`for`, `defer`) non gardée en
  profondeur → SIGSEGV natif sans P1029.** `vader/parser/parser.vader:2188,2333`.
  `MAX_EXPR_DEPTH` ne garde que `parse_expr`/`parse_type`. `for { for { … } }`
  profond ou `defer defer …` déborde le stack C (exit 139) sans diagnostic. **Fix**
  : incrémenter `expr_depth` dans `parse_block` (le choke-point).

- **comptime-1 — un char dans une interpolation `@comptime` rend le numéro de
  codepoint, pas le glyphe.** `vader/comptime/eval_lit.vader:81`. `interp_value`
  rend `CharValue` via `"${x.value}"` (le `u32`). Repro : `@comptime GREETING ::
  "hi ${'A'}"` bake « hi 65 », alors que le même code runtime donne « hi A ».
  **Fix** : encoder le codepoint en UTF-8 comme `Display` runtime.

- **lexer-1 — un commentaire de ligne ne se termine pas sur un CR seul → avale du
  code (fichiers Mac classique).** `vader/lexer/lexer.vader:861`. `advance` compte
  un `\r` seul comme fin de ligne (avec test dédié), mais le scan de commentaire
  s'arrête seulement à `\n`. Repro : `// c<CR>x = 42` → `x = 42` avalé, résultat
  99 au lieu de 58, VM et natif. **Fix** : arrêter la boucle aussi sur `\r`.

### Lower / resolver / comptime

- **lower-desugar-3 — l'analyse de closures saute les corps de const top-level →
  local capturé d'un const-block mal-lowéré.** `vader/lower/closure_analysis.vader:
  44-63`. `analyze_module` ne visite que `FnDecl`/`ImplDecl` ; un const initialisé
  par un block qui déclare des locals et retourne une lambda capturante n'a pas
  d'entrée `lambda_captures` → cell non promue → symbole absent après lifting.
  Repro : `make_adder :: { base :: 10 ; (n) -> base + n }` → VM ICE local.set,
  natif `unreachable` exit 134. **Fix** : walker aussi les `ConstDecl`.

- **resolver-modules-2 — les imports namespace-alias contournent la visibilité
  d'export.** `vader/resolver/wire.vader:76`, `typecheck/orchestrate.vader:1428`.
  Un import destructuré fire R2008 sur un symbole privé ; `P :: import "m"` n'a
  aucun check → `P.secret` atteint un `secret` non-`export`. Repro : module `lib`
  avec `secret` privé, `L :: import "snippet/lib" ; L.secret()` → imprime `42`,
  exit 0. **Fix** : filtrer par `Visibility.Private` dans `build_namespace_exports`
  / `build_namespace_type_members`.

- **resolver-modules-3 — détection de doublon fn-vs-import dépendante de l'ordre
  des décls.** `vader/resolver/collect.vader:117,198`. Import d'abord : `add_fn`
  écrase le slot en silence. Fn d'abord : `bind_import` fire R2011. Le même
  programme compile ou échoue selon l'ordre. **Fix** : politique symétrique.

- **comptime-3 — un `@assert` sur un const `@comptime` est silencieusement
  ignoré.** `vader/comptime/assert.vader:28`, `check.vader:41`.
  `evaluate_assert_decls` reçoit un `bindings` vide → l'ident du const n'est pas
  résolu → résultat non-`BoolValue` → C4015 jamais émis. Repro : `@comptime LIMIT
  :: 10 ; @assert(LIMIT == 999)` compile clean exit 0. **Fix** : threader
  `program.comptime_decls` dans le contexte d'assert.

### Backend intermédiaire — unions fermées sans garde

- **midir-1 — `build_stmt` drop silencieusement `LoweredYield`/`LoweredAwait` ;
  `emit_instr` drop toute `Instruction` non gérée.** `vader/midir/build.vader:
  427-495`, `emit.vader:954-1350`. If-chains sur unions **fermées** sans else/panic
  final, alors que le backend bytecode `panic`e (`emit_body.vader:60`). Un marqueur
  coroutine survivant au lowering → miscompile silencieux du **chemin de prod**
  (midir). Viole la règle CLAUDE.md « exhaustif ou `@partial` ». **Fix** : `match`
  exhaustif avec `panic` sur les marqueurs.

- **lower-mono-2 — deux détecteurs de type-param divergents ; `type_contains_param`
  omet `UnionType`.** `vader/lower/lower_generic_call.vader:423`,
  `lower_mono_struct.vader:159`. `has_type_param` (exhaustif) descend dans les
  unions ; `type_contains_param` (`@partial`) traite `T | null` comme concret →
  `all_concrete` pourrait émettre un mangle avec un `T` non résolu, désynchro des
  clés de mangle. Latent (l'inférence lie le param avant). **Fix** : un seul
  détecteur exhaustif.

### LSP / CLI / diagnostics

## Features DX proposées

1. **Assignation composée bit-à-bit / shift `&= |= ^= <<= >>=`** (syntax-spec-dx-4).
   Coût minime : 5 tokens lexer + 5 arms dans `compound_op` (desugar générique
   `lhs = lhs op rhs`, aucun changement typecheck/lower) + printer + SPEC.
   Bénéfice réel et *self-serving* : le code bit-heavy du compilateur (masques
   lexer, crypto, `std/core/bigint`) écrit `x = x | mask` / `x = x << 4`.

2. **Fold de cast numérique au comptime** (comptime-7) : `f64(SOME_INT)` /
   `i32(3.9)` en `@comptime` devraient folder au lieu de C4001. Touche
   `eval_call.vader` seul.

3. **Lint « wildcard sur union fermée qui renvoie une valeur »** — le footgun
   documenté dans CLAUDE.md (silence T3013) mériterait un warning dédié ciblant
   `Type`/`BcType`/AST `Expr`/`Stmt`/opcodes, pour transformer les `_ ->`
   dégradants (midir-1, lower-mono-2) en erreurs conscientes.

4. **Préallocation de tableau (`with-capacity`/`fill`)** — trou DX connu (mémoire
   projet), non re-couvert ici mais toujours ouvert.

## Réfutés (vérifiés, écartés)

10 constats REFUTED par la passe adversariale — utile de savoir ce qui est **sain** :

## Priorités recommandées

1. **Correctifs ~1 ligne à très haut rendement** : CR-a (`uint32_t` cible F64),
   CR-b (`ArraySlice` dans `op_allocates`), CR-c/CR-d (`emit_postfix_target` /
   arm `AwaitExpr`), CR-g (vues octets dans `compare_ascending`), midir-1
   (`panic` sur marqueurs coroutine).

2. **Soundness typecheck (famille de trous jumeaux)** : porte de compatibilité
   d'opérande unique partagée par `check_equality`/`check_compare`/`check_arith`
   (CR-k, tc-core-1/2, tt-2), validation des casts (tt-4, T3010), plage des
   littéraux entiers (dx-1/tc-core-4), invariance des conteneurs mutables (CR-j,
   T2).

3. **`?` (try)** : drainer les defers (CR-e), vérifier le contrat de propagation
   + type de retour (lower-desugar-2, dx-6) — ce dernier crashe le compilateur.

4. **stdlib entrée-adverse** : regex profondeur + clôture epsilon itérative
   (CR-h/CR-i, CR5), json NaN/Inf + dtoa non-fini (dom-6), json leading-zero
   (dom-4), time iso8601 (dom-5).

5. **fmt sûr** : les trois bugs de round-trip `?`/`await` (CR-c/CR-d) + fmt-4
   (bases enum), puis fuzz format→reparse sur tout l'arbre.

6. **Nettoyage structurel à fort ROI** : supprimer l'émetteur tree-walker mort
   (bytecode-1, ~1000 LoC) et `load_module_files` (resolver-modules-4, landmine
   node-id) ; collapser les paires de tables synchronisées main (mangle-suffix,
   primitive-tag, containment LSP, renderers de diagnostics) ; un seul walker de
   substitution exhaustif (tt-5, + arm `DistinctType`).

7. **Chantiers de fond** : IR verifier ciblant les analyses de flux sur
   locals mutables pré-SSA (midir-2, const_fold), unification des conventions
   d'erreur stdlib (core-3, domain-7), lint wildcard-sur-union-fermée.
