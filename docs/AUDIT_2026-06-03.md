# Audit du compilateur Vader + stdlib (2026-06-03)

> Audit mené via workflow multi-agents (53 agents) : survol par sous-système
> (20 unités), 6 lentilles transverses (maintenabilité, DX, évolutivité, code
> mort, features, migration→stdlib), puis vérification adversariale des
> findings prioritaires. Pipeline : 229 findings bruts → 77 transverses → 76
> dédupliqués → top 26 vérifiés (25 confirmés, 1 réfuté), 50 en annexe non
> vérifiée. Le TS jetable (`src/`) est exclu du périmètre.

## Synthèse

L'arborescence `vader/` est globalement saine et bien structurée par
responsabilité, mais l'audit fait ressortir **cinq thèmes récurrents** :

1. **Absence de visiteur AST partagé** — la même descente récursive sur l'union
   fermée `Expr`/`Stmt` est recopiée à la main dans 8+ fichiers (LSP, comptime,
   resolver). Plusieurs copies finissent par `_ -> {}` : une nouvelle variante
   d'AST cesse silencieusement d'être traversée — exactement le mode d'échec
   contre lequel §6/§9 mettent en garde.
2. **Trous d'extension dans la stdlib qui font fuiter des internes partout** —
   `std/collections` n'expose aucun constructeur (`MutableMap { .buckets = [], .size = 0 }`
   recopié dans **76 fichiers**), `std/json` n'expose aucun accesseur (re-dérivé
   dans 7+ fichiers LSP), `std/string` n'a ni `join` ni `byte_substring`.
3. **Unions fermées avec passthrough dégradé** — `remap_instr`/`with_dst`
   (midir/cfg_dce) reconstruisent l'union 27-variantes `Instruction` en chaînes
   `if` terminées par `return ins`, neutralisant T3013.
4. **DX d'utilisation du langage faible sur deux surfaces clés** — diagnostics
   réduits à une ligne (pas de snippet source ni de caret), `base64::decode`
   corrompt tout octet ≥ 0x80, LSP qui re-typecheck tout le projet depuis le
   disque à chaque frappe en ignorant le buffer mémoire (bug de correction sur
   fichier non sauvegardé).
5. **Monolithes 3-8× au-delà de la cible §7** — `lower_expr.vader` (4079),
   `parser.vader` (3249), `lower.vader` (2733), `orchestrate.vader` (1659) —
   découpage en partie bloqué par la limite mutual-import déjà documentée.

## Quick wins (effort small, impact ≥ medium)

- **`base64::decode` corrompt le binaire** — bug de correction, fix local,
  miroir d'un pattern existant (`crypto.vader`). À faire en premier.
- **Ajouter `std/string::join`** — inverse manquant de `split`, supprime
  `join_commas`/`join_lines`/`join_offsets` (4 sites).
- **Consolider les 4 copies de `mangle_module_id`** — élimine un footgun de
  drift de mangling documenté (classe de bug parité §9).
- **Bug LSP buffer-périmé** — thread `state.docs` dans `load_project`
  (indépendant du travail de cache perf).
- **`new_map`/`new_set` dans `std/collections`** — point d'extension unique,
  déverrouille le travail resize/insertion-order différé.

## 1. Maintenabilité

**1. `remap_instr` / `with_dst` : matches exhaustifs sur `Instruction` + rebuild de champs partagé**
— `vader/midir/cfg_dce.vader:349` (147 LoC) et `:509` (69 LoC). Deux chaînes
`if` terminées par `return ins` sur l'union fermée 27-variantes `Instruction`.
C'est l'**unique** site qui remappe `dst` dans `compact_locals` (pas de backstop
`for_each_write_local`) : une future variante porteuse de `LocalId` passerait à
travers `return ins` et corromprait l'IR compacté sans erreur. Les frères
`for_each_read_local`/`dst_of` (`analyses.vader:63`) sont déjà exhaustifs.
**Action** : convertir en `match ins { is X -> ... }` exhaustif (variantes sans
`dst` → arm `@partial` + `// why:`) ; factoriser le rebuild dans
`map_instr_locals(ins, m: i32[])` — passer le `i32[]` directement, **pas** une
closure (les closures à 2 free-var segfaultent sur le VM self-host,
TODO.md:855). Auditer aussi `instruction_has_side_effect` (`analyses.vader:46`,
`_ -> false` non marqué). *(high / medium / high)*

**2. Helpers de copie `with_*` pour `CFGFunction` et `BcFunction`**
— `CFGFunction` reconstruit champ par champ à 5 sites (`cfg_dce.vader:92/186/318`,
`const_fold.vader:66`, `escape.vader:191`), `BcFunction` respellé à 8 sites
(`emit.vader:156/165/588/733/777/859/1058`, `slot_coalesce.vader:200`). Plus
grosse dette de respelling dans midir/bytecode. **Action** : ces fichiers
partageant le module `"vader/midir"`, ajouter des helpers de copie intra-module
dans `cfg.vader` ; ajouter un champ passe de 13 sites à 1. *(high / medium / high)*

**3. Un visiteur AST `Expr`/`Stmt` partagé — 8+ walkers recopiés à la main**
— `comptime/deps.vader:220`, `check.vader:345`, `lsp/indexer.vader:501`,
`hover.vader:295`+`:405`, `signature_help.vader:143`, `inlay_hint.vader:129`,
`parser/assign_ids.vader:184`, `resolver/body.vader:382`. 4 sites finissent par
`_ -> {}` : une nouvelle variante `Expr` n'y descend plus silencieusement.
`assign_ids.vader:184` est l'énumération exhaustive de référence. **Action** :
`for_each_child_expr`/`for_each_child_stmt` dans `vader/parser`, modelé sur
`midir/lowered_walk.vader::for_each_reference`, callback en champ `fn`-type
**inline** (pas un alias — le typechecker self-host mis-infère les champs fn
aliasés en `?|void`, cf. `lowered_walk.vader:52-57`). Les deux find-position
walkers exigent un visit **pré-ordre**. Per §8, à revoir avec l'utilisateur
avant migration (contrat de traversée cross-pass). *(medium / large / high)*

**4. Découper les 4 pires monolithes — en reconnaissant le blocage mutual-import**
— `lower_expr.vader` (4079), `parser.vader` (3249), `lower.vader` (2733),
`orchestrate.vader` (1659). **(déjà suivi en partie)** Le découpage traverse des
cycles de récursion mutuelle documentés — même limite que pour `emit_body.vader`
(`GAPS.md:119`), contournée ailleurs via callbacks (`lower_match.vader:28`,
`lower_try.vader:18`). **Action** : (a) gains hors-cycle faisables : extraire
`lower_call` + sondes `try_lower_*` (`lower_expr.vader:745-2020`, où se
concentrent les bugs parité f64.lt/GATE-B/pick_overload) dans `lower_call.vader`
(sens unique, un callback) ; sortir les helpers de mangle + walker DCE
(`lower.vader:2055+`). (b) Traiter les splits `parser`/`orchestrate` comme
**gatés** sur le travail forward-decl / per-file-cap (GAPS.md:119) — item à plus
fort levier à planifier. *(medium / large / high)*

## 2. Developer experience (DX)

**1. LSP : re-typecheck du projet entier depuis le disque par requête, en ignorant le buffer mémoire**
— `lsp/hover.vader:183-185`, `inlay_hint.vader:39-41`, `signature_help.vader:37-39`,
`diagnostics.vader:33-35`. Chaque handler appelle `load_project(file_path)` +
`check_project_with_bodies(...)`, re-parsant ~30 modules stdlib par requête
(~2-3 s, TODO.md:1017). Pire : lit `file_path` **depuis le disque**, ignore
`state.docs` (`state.vader:79-96`) → hover/diagnostics reflètent le dernier
fichier **sauvegardé**, pas le buffer éditeur. **Bug de correction**, pas qu'un
problème de perf. **Action** : (1) prioritaire — overlay mémoire (uri→texte
depuis `state.docs`) consulté avant le disque dans `read_file` (`loader.vader:215`,
`discover.vader:105`) ; (2) perf (moitié suivie) — cacher `TypedProgram`+projet
keyé par uri+hash, invalidé sur didChange. *(high / medium / high)*

**2. Renderer de diagnostic avec snippet source + caret — la surface utilisateur la plus faible**
— `diagnostics/diagnostic.vader:141-154` (`render`) et `lexer/dump.vader:232`
(`format_diagnostic`, le vrai chemin compilo). Les deux ne produisent qu'une
ligne `error[T3001]: msg at file:12:5` et ne lisent que `d.primary.start` ;
`Span.end` est du poids mort. Aucun caret nulle part. Source joignable via
`SourceFile.content` (`resolver/module.vader:43`). **Action** :
`render_with_source(d, source)` découpant la ligne fautive + underline `^^^` sur
`start..end`, câblé dans `format_diagnostic`/`dump_diagnostics` via
`print_pipeline_diags` (`cli/main.vader:659`) — **pas** `render`. Par ailleurs :
le flag `--diagnostics=text|json` est stocké (`main.vader:138`) mais jamais lu →
no-op silencieux, à implémenter ou supprimer. *(high / medium / high)*

**3. Tests colocalisés pour les cœurs non testés : lsp/vm/fmt/midir/c_emit ≈ 0, parser quasi-0**
— `lsp/` (0/17), `vm/` (0/6), `fmt/` (0/5), `midir/` (0/10), `c_emit/` (0/2),
`parser/` (1/5) ; typecheck à 23/40 par contraste. **(déjà suivi en partie)**
`IMPROVEMENT.md` item 6 ne suit que resolver/typecheck. MEMORY note que la suite
ne valide jamais l'emitter Vader — pourquoi §9 cogne sur des bugs de mis-emit
latents. **Action** : prioriser le chemin emitter — `c_emit/body.vader::sanitise`
+`c_string_lit`, `vm/builder.vader`, disambiguation parser (`a < b > c` vs
`Foo<Bar>`), `lsp/signature_help::active_param_index`,
`fmt/printer::compute_decl_alignment_widths`. *(high / medium / high)*

**4. `base64::decode` corrompt le binaire — octets ≥ 0x80 UTF-8-encodés**
— `stdlib/std/base64/base64.vader:79-139`. `decode` accumule via
`sb.append_char(char(byte))` → `append_char` fait `self.parts.push("${c}")` qui
UTF-8-encode le codepoint → tout octet ≥ 0x80 émet 2 octets, round-trip lossy.
Les tests ne couvrent que de l'ASCII. **Action** : remplacer le `StringBuilder`
par `out: u8[] = []`, push `u8(...)` aux 4 sites, `return out.as_string()`.
Miroir de `crypto.vader:188-202`. Ajouter un test round-trip non-ASCII.
*(high / small / high)*

## 3. Évolutivité / extensibilité

**1. C-emit alimenté par le tree-walker abandonné, alors que le chemin CFG validé ne sert qu'au dump `.virt`**
— `cli/main.vader:582` (`run_c_stage` → `emit_project`) vs `:545/568`
(`emit_bytecode_from_cfg`). `main.vader:556-560` documente que `emit_project`
« élide les temps 3-adresse et diverge sur ~150 snippets de parité ». §9
prévient que le backend C lit la **même** table `BcType` → la qualité du codegen
C est épinglée au chemin documenté comme divergent. **Action** : router
`run_c_stage` par `build_cfg_project → eliminate_dead_cfg → annotate_escape →
emit_bytecode_from_cfg`, supprimer `emit_project` + helpers (~600 LoC).
Architecture-level — à revoir per §8 avant de lander.
*(high / large / high — non re-vérifié indépendamment)*

**2. `MutableMap` : le littéral interne `{.buckets=[], .size=0}` fuit dans 76 fichiers**
— `stdlib/std/collections/collections.vader:50` (aucun constructeur exporté).
Tout map/set vide épelle la représentation privée → ajouter/renommer un champ
(load-factor/resize, ou vecteur d'ordre de clés pour JSON déterministe) est une
édition shotgun multi-fichiers. **Action** : `new_map`/`new_set` (§6) — point
d'extension unique. *(high / medium / high — count de 76 fichiers confirmé)*

## 4. Code mort / à supprimer

**1. Collapser les 4 copies de `mangle_module_id`**
— `lower/lower.vader:1082`, `lower_expr.vader:1765`, `lower_expr.vader:1321`
(`mangle_module_id_local`, renommé juste pour esquiver la collision same-module),
`cli/main.vader:599-611` (`entry_main_name` inline le même char-walk). Quatre
forks byte-équivalents (`'/'`→`'_'`, `'.'`→`'_'`). Le drift de mangling est une
classe de bug parité documentée ; la copie `entry_main_name` est ce dont dépend
la directive VM `export main <index>` — un changement de schéma lowerer désync
silencieusement la copie cli (retourne `""` sans diagnostic). **Action** :
`export mangle_module_id` dans `vader/lower/helpers.vader` ; supprimer les 2
doublons + le `_local` ; réécrire `entry_main_name` en
`"${mangle_module_id(tp.resolved.display_path)}$main"`. *(medium / small / high)*

> *Réfuté : « supprimer `peephole.vader` + `write_virt` » — cibles de régression
> vivantes dans `tests/snippets/peephole_rules/` et `builder_roundtrip/`.*

## 5. Nouvelles fonctionnalités à ajouter

**1. `std/string::join` — l'inverse manquant de `split`**
— pas de `join` dans `string.vader` (`split` à `:385`). Le compilateur embarque
un `join_commas` one-off (`typecheck/field.vader:461`, `struct_lit.vader:122`) =
exactement `join(names, ", ")`. **Action** :
`export join :: fn(parts: string[], sep: string) -> string`, colocalisé avec
`split`. Rerouter `join_commas`, `join_lines` (`lsp/hover.vader:161` **et**
`lower/dump.vader:442`, doublon), `join_offsets` (`c_emit/emit.vader:422`). Ne
**pas** y plier les joins sur `Type[]` via callback. *(medium / small / high)*

**2. `new_map`/`new_set` dans `std/collections`**
— `export new_map :: fn<K: Hash & Equals, V>() -> MutableMap(K, V) = MutableMap { .buckets = [], .size = 0 }`
+ `new_set`, puis migrer. Nettoie `ctx.vader`, `dce.vader`, `orchestrate.vader`,
le constructeur `ResolvedModule`, `json`, `cli` simultanément.
*(high / small / high)*

**3. Accesseurs `std/json` (`get`/`as_string`/`as_number`/`as_bool`/`path`)**
— `json.vader:25-45` (seulement types + parse/stringify). Lire un doc parsé
exige `match` puis indexer le champ privé `entries: MutableMap` et re-`match`.
Chaque fichier LSP réimplémente `get_field`/`get_string`/`get_number` —
duplication verbatim sur 7 fichiers (~250 LoC). *(high / medium / high)*

**4. `byte_substring` dans `std/string`**
— aucun slice par offset octet. Chaque consommateur re-dérive
`s.bytes()[start..<end].as_string()` : ~38 occurrences dans `vm/parser.vader`
seul, + `lsp/indexer.vader:459/654`, `fmt/printer.vader:79`. Opération string la
plus dupliquée. **Action** :
`export byte_substring :: fn(s: string, start: usize, end: usize) -> string`.
*(high / medium / high)*

**5. Le formatter ne peut pas wrapper les appels/struct-lits/arrays trop longs**
— `fmt/printer.vader:1239-1258` (`emit_call` toujours single-line), `:1557-1595`.
**(déjà suivi — fmt in-progress)** MAX_LINE n'est consulté que pour le layout
union de `emit_const_decl` (`:851`). Gap cœur empêchant fmt d'être un vrai
formatter. **Action** : wrapping piloté par largeur via `render_expr_for_measure`
+ MAX_LINE. *(high / large / high)*
